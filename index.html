Timely information
Meaningful data at the moment you need it most

Monitoring must give you valuable, timely and relevant information, not just be a burden and whine so frequently that it ends up being like the “boy who cried wolf”.

The on-call engineer from the appropriate team should get a single aggregated message with valuable insight like: “25 servers are failing because the v_X storage volume is offline, this will impact the X business service within 2HRS, talk to Y manager to arrange for a fix and check ticket 123 for troubleshooting instructions”.

The side benefit?

Well, you get automated infrastructure documentation in an open format that is easy to use, all in one place, including an "exploration"/"visualization" dashboard, a RESTful API and an advanced query-building console to create all sorts of dynamic reports.

Effortlessly manage complexity

Forget about documenting all of that using different systems with disparate data formats, spreadsheet files sent over email, horrible tables inside powerpoint presentations and network diagrams that are impossible to keep up-to-date automatically.
