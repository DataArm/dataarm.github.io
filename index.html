We live in a different era
  • The IT world is changing drastically, DevOps/Agile is shifting our focus to continuous integration, rapid deployments, automation, cloud, IoT, IaaS, PaaS, SaaS, containers, etc.
  • This is a good thing, but somehow it feels that you are now forced to manage an ever growing, more complex number of services, platforms, and technologies, while simultaneously expected to be faster than ever without letting go off legacy systems.
Timely information
You work faster and more efficiently, but

You feel the pain when documentation and configurations starts to lag behind:

  • IT infrastructure changes constantly, how up-to-date are those monitoring configurations, inventory reports and network diagrams?
  • When a monitoring alert is triggered, how long does it take to find out where is the server, which are the impacted services and their stakeholders?
  • Software and hardware need to be replaced much more frequently, how fast can your staff generate the list of connected services to be certain of the dependencies when planning for a "maintenance window"?

You need valuable, timely, accurate and relevant information, it used to be as simple as picking up a couple of ITSM tools, it is not enough anymore:

  • The on-call engineer needs a single, timely, aggregated message with valuable insight like: “25 servers are failing because the v_X storage volume is offline, this will impact the X business service within 2HRS, talk to Y manager to arrange for a fix and check ticket 123 for troubleshooting instructions”
  • You can't afford the significant amount of time wasted trying to find where to start looking for the root cause, at the most critical moment.
Complexity increased
Complexity exploded

From our experience, you and your team have now anywhere between 50,000 and 250,000 "things" to manage: servers, virtualization platforms, storage, routers, IP networks, DNS/DHCP, load-balancers, organization charts, ACL's, appliances, vendors, data centers, racks, clusters, containers, cloud providers, environments, etc.
Forget about documenting all of that using legacy systems with disparate data formats built for the static, slow changing IT world of the nineties.
