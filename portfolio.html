Visualizing impact analysis

  • Multiple monitoring systems send 2,500+ alerts during 5 minutes
  • Backend server uses a graph database to de-duplicate alerts, analysis impact and produce a single aggregated message for escalation
  • The resulting impact analysis graph is pushed via web sockets in real-time (efficient bandwidth usage by only sending changes which can scale to hundreds of simultaneously subscribed clients)

Interactive globe/dashboard

The dashboard pulls data from Salesforce and helps to execute tests and understand how failures on any of the 1,400+ DNS Top level domains are geographically localized, and which population centres may be impacted the most:

  • Monitoring centers (Geolocation data) as green or red dots (real-time test results)
  • Dark blue circles represent population density (public data)
  • The darker a country's color is, the more it depends on internet (public data)
  • Submarine cables hint on internet exchange points (topography data)

Alerts on external events

Example of integrating an external Publish/Subscribe system (Amazon's AWS SNS/SQS) with an on-premise hosted application.

Received events on the backend server are pushed themselves through web sockets into the browser (scales up to hundreds of clients and can be used as a proxy on different campuses).

DNS server oversubscription

For the New gTLDs program, this interface took data from Salesforce in a column/row format and from Root DNS public infrastructure (Zone AXFR) into a graph database to be able to show how DNS servers were shared amongst TLDs from the same backend operators.

The goal was to visualize the level of "connectedness" of DNS infrastructure and how some legal entities were "outsourcing" the operation to the same providers.

Optimal route calculation

The map shows the location of rubbish cans (geolocation API overlayed on Google Maps' data) and uses a graph database to calculate the optimal path for the collection truck to only visit streets with full bins.

Path calculation uses proximity to clustered nodes as well as the weight of paths based on predicted traffic.

The blockchain is used to calculate wallet balances based on recycling goals.
