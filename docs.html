Database access

You can have console access to the database by going to your dashboard's:

URL/#database

Run the following command to start an interactive introduction:

:play intro

Python, .NET, Java and Javascript are supported natively and you can programatically connect to the database and unleash the potential for any integration you can imagine!

As an example, let's assume we have some nodes connected like this:

(a:Service)-[r:DEPENDS]->(b:DNS_Label)

And there is an "url" property and a "Do_Not_Monitor" flag on "r"

We could filter the data and then do something with the results, like populating our external monitoring SaaS (Site24x7, Uptrends!, etc.), here is how to get started with Python:

#!/usr/bin/env python

from neo4j.v1 import GraphDatabase, basic_auth, StatementResult

_db = GraphDatabase.driver(
    "bolt://{hostname}",
    auth=basic_auth('{user}', '{password}')
)

# Session wrapper
def run_query_get(query):
    with _db.session() as _sess:
        return _sess.run(query)

# Main code
for r in run_query_get('MATCH (a:Service)-[r:DEPENDS]->(:DNS_Label) WHERE NOT EXISTS(a.Do_Not_Monitor) AND r.url =~ "(?s)^http.*" RETURN a.name AS name, r.url AS url'):
    print "Do something with URL %s and the name of the service %s" % (r['url'], r['name'])

As simple as that! Here are the full documents about the syntax for queries (called "Cypher"), and here is the documentation for the driver specific configuration and supported programming languages (we only support the "BOLT" protocol by default)

Agent

The agent is a binary executable that creates a JSON output that contains:

  • Hostname
  • Boot time
  • Currently logged users
  • Listening network sockets (and process)
  • Established network connections (and process)

You can download the agent from your dashboard, e.g. on CentOS7:

curl -O URL/repo/inventory/CentOS/7 && chmod +x ./inventory

You can view a "pretty" output of the output by running:

./inventory | python -m json.tool

Just configure a scheduler to run the binary at regular intervals pointing to your dashboard's "/api/inventory" entry point.

As an example, on Linux:

./inVentory --target URL/api/inventory

You can use a different tool to create the data and POST to the same API entry point
